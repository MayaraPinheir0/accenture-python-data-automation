## 2. Estrutura do Repositório

Entenda a função de cada arquivo neste projeto de automação. Cada script representa um degrau na construção da lógica de programação.

### 🧩 Módulo 1: Operadores & Base
* `01_syntax_and_variables.py`: **O Átomo.** Declaração de variáveis e entrada/saída de dados (`input`/`print`).
* `02_data_types.py`: **A Natureza do Dado.** Manipulação de Inteiros, Floats e Strings (Textos).
* `03_type_casting.py`: **Transformação.** Como converter tipos de dados para garantir que a lógica de cálculo funcione.

### ⚖️ Módulo 2: Condições & Repetição
* `04_conditionals.py`: **Tomada de Decisão.** Uso de `if`, `elif` e `else` para criar caminhos lógicos no software.
* `05_loops_and_iteration.py`: **Automação de Ciclos.** Uso de `for` e `while` para processar informações repetitivas sem duplicar código.

### 🗃️ Módulo 3: Collections (Estruturas de Dados)
* `06_lists_and_sequences.py`: **Arrays Dinâmicos.** Como gerenciar listas de itens que podem ser alteradas (mutáveis).
* `07_tuple.py`: **Dados Imutáveis.** Proteção de informações que não devem ser alteradas durante a execução.
* `08_dictionary.py`: **Mapeamento Chave-Valor.** Organização de dados complexos (ex: perfis de usuários ou tabelas de banco de dados).

## 4. Informações Técnicas & Comparações 
*Clique nos tópicos abaixo para ver as análises de cada conceito.*

<details>
  <summary>🧱 <b>Tipos Primitivos vs. Collections</b></summary>

| Categoria | Exemplo | Comportamento |
| :--- | :--- | :--- |
| **Tipos Primitivos** | `int`, `str`, `float` | Guardam um **único valor** na memória por vez. |
| **Collections** | `list`, `dict`, `tuple` | Estruturas que agrupam **múltiplos valores** de forma organizada. |

</details>

<details>
  <summary>🔁 <b>Controle de Fluxo: for vs. while</b></summary>

> **Padrão de Projeto:** Escolha o loop baseado na previsibilidade da repetição.

* **`for`**: Ideal quando o número de iterações é **conhecido** ou limitado a uma sequência.
* **`while`**: Ideal quando a repetição depende de uma **condição externa** ou desconhecida (ex: aguardar resposta do servidor).

</details>

<details>
  <summary>📦 <b>Mutabilidade: Lista vs. Tupla</b></summary>

* **`list` (Mutável)**: Permite alteração, adição e remoção. Funciona como uma pasta de arquivos que você sempre atualiza.
* **`tuple` (Imutável)**: Protege a integridade dos dados. Uma vez criada, é "escrita em pedra" (ex: dias da semana ou constantes do sistema).

</details>

<details>
  <summary>🗂️ <b>Acesso à Informação: Lista vs. Dicionário</b></summary>

* **`list` (Sequencial)**: O acesso é feito pela **posição** (Índice). Útil quando a ordem importa.
    * `lista[0]` → Retorna o primeiro elemento.
* **`dict` (Associativo)**: O acesso é feito por uma **etiqueta** (Chave). Útil quando o contexto do dado é o mais importante.
    * `dict["email"]` → Retorna o dado associado àquela chave específica.

</details>
